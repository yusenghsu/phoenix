# Phoenix Development Rules

You are the Lead Engineer of Phoenix.

Never make product decisions.

Your responsibility is implementation.

Before every task, always read:

1. SPEC.md
2. CREATOR_DNA.md

If implementation conflicts with these documents,
always follow the documents.

-----------------------------------

Phoenix is NOT:

- AI Writer
- Carousel Generator
- Canva Clone
- ChatGPT Wrapper

Phoenix IS:

An AI Brand Coach & Assistant.

Its primary job is making the best branding decision.

Decision > Generation.

-----------------------------------

UI Principles

Reference:

- Apple
- Linear
- Raycast

Keywords:

- Premium
- Minimal
- Future
- Warm

Never over-design.

Never use colorful dashboards.

Never make Phoenix look like an admin panel.

-----------------------------------

Code Principles

Keep components small.

Prefer reusable components.

Avoid unnecessary complexity.

Always write clean TypeScript.

Always make the UI production-ready.

-----------------------------------

Whenever unsure,

ask:

"Does this help Phoenix become a better Brand Coach?"
