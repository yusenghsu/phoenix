# Roadmap
## Phoenix — 開發里程碑

---

### 開發哲學

**Ship, Learn, Iterate。**

不追求完美的 v1，追求可以驗證核心假設的最小版本。
每個 Phase 結束後，根據真實使用者回饋決定下一步。

**核心假設**（需要 Phase 1 驗證）：
1. 使用者願意把「今天要發什麼」交給 AI 推薦
2. 品牌記憶讓內容品質明顯優於通用 AI 工具
3. 一鍵產生 + 複製到 Canva 的流程夠順暢

---

### Phase 0：現狀（已完成）

> 一個靜態的 AI 輪播貼文生成器，使用者手動輸入主題。

**已有**：
- Next.js + TypeScript + Tailwind + shadcn/ui 架構
- OpenAI gpt-4o 串接
- 輪播內容產生（封面、頁面、CTA、Caption、Hashtag、Canva 文字）
- 輪播視覺預覽 + 一鍵複製

**缺少**：
- 品牌記憶（每次都要重新說明自己是誰）
- 主動推薦（使用者自己想主題）
- 發布追蹤（沒有學習閉環）

---

### Phase 1：Brand Memory MVP（目標：2 週）

**目標**：讓 Phoenix 記住使用者是誰，產生的內容明顯更個人化。

**交付物**：

- [ ] **Onboarding 流程**（3 步驟問卷）
  - 品牌身份設定（職業、受眾、目標）
  - 品牌聲音設定（語氣選擇 + 參考貼文上傳）
  - 生成「品牌身份卡」預覽

- [ ] **Brand Memory 儲存**
  - Phase 1 使用 localStorage（無需後端）
  - 定義完整的 BrandProfile TypeScript 型別

- [ ] **品牌記憶注入 Content Agent**
  - `buildCarouselPrompt()` 改為接受 BrandProfile
  - System prompt 包含品牌身份 + 聲音 + 近期主題

- [ ] **`/brand` 設定頁面**
  - 查看與編輯所有品牌記憶
  - 重置功能

**驗證指標**：
- 用品牌記憶產生的內容，使用者自評「這聽起來像我說的話」比例 > 70%

---

### Phase 2：主動推薦系統（目標：Phase 1 後 2 週）

**目標**：Phoenix 每天主動推薦主題，使用者不再需要空白輸入框。

**交付物**：

- [ ] **首頁重設計**
  - 移除「輸入主題」為主要入口
  - 新增「今日推薦」區塊（3 張主題卡片）
  - 每張卡片顯示：主題、推薦理由、格式建議

- [ ] **Strategy Agent**（`/api/recommend`）
  - 根據品牌記憶 + 模擬趨勢資料產生推薦
  - Phase 2 暫不接真實外部 API，由 AI 模擬

- [ ] **發布追蹤**
  - 「標記已發布」按鈕
  - 發布記錄存入 localStorage
  - Strategy Agent 讀取發布記錄避免主題重複

- [ ] **Today's Brief**
  - 首頁頂部簡短摘要：你最近發了什麼、距離上次發文幾天

**驗證指標**：
- 使用者連續 7 天使用 Phoenix 推薦的主題比例 > 50%

---

### Phase 3：真實趨勢整合（目標：Phase 2 後 3 週）

**目標**：Trend Agent 接入真實外部資料，推薦品質大幅提升。

**交付物**：

- [ ] **Trend Agent**（`/api/trends`）
  - 整合 Google Trends API（台灣地區）
  - PTT 理財版 / 保險版熱門討論爬蟲（或 RSS）
  - 每日 Cron Job 定時執行

- [ ] **後端資料層**（從 localStorage 升級）
  - 選型：Supabase（PostgreSQL + Auth）
  - 建立 `brand_profiles`、`content_records` 資料表
  - API Route 改為讀寫資料庫

- [ ] **多用戶支援**
  - 加入 Auth（Supabase Auth 或 Clerk）
  - 品牌記憶與發布歷史綁定使用者帳號

**驗證指標**：
- 使用者反映「今天推薦的主題，剛好是我也有注意到的時事」比例 > 60%

---

### Phase 4：內容格式擴展（目標：Phase 3 後 1 個月）

**目標**：超越輪播，支援更多內容格式。

**交付物**：

- [ ] **單張圖文**：一句話金句 + 設計建議
- [ ] **Reels / IG 短影音文案**：腳本格式，分鏡說明
- [ ] **留言回覆建議**：針對常見留言提供標準回覆模板
- [ ] **LINE 貼文版本**：針對 LINE 受眾優化的純文字版本
- [ ] **內容日曆視圖**：月曆顯示已發布和已產生的內容

---

### Phase 5：效果學習與個人化（目標：長期）

**目標**：Phoenix 越用越懂你，推薦越來越準。

**交付物**：

- [ ] **Instagram API 串接**：自動讀取互動數據（需 Meta 開發者審核）
- [ ] **效果分析頁面** `/insights`：哪類主題帶來最多互動、最多詢問
- [ ] **個人化互動潛力預測**：根據你的帳號歷史，預測每個主題的預期互動
- [ ] **A/B 主題測試**：同一概念產生兩個不同角度，協助使用者選擇

---

### 技術債追蹤

開發過程中需要在合適的時機處理：

| 項目 | 優先級 | 目標 Phase |
|------|--------|-----------|
| 現有 prompt 邏輯與品牌記憶解耦 | 高 | Phase 1 |
| 串流輸出（避免長時間等待） | 中 | Phase 2 |
| 錯誤邊界與 loading skeleton | 中 | Phase 2 |
| API Rate Limit 處理 | 高 | Phase 2 |
| 響應式設計行動端優化 | 中 | Phase 3 |
| OpenAI 成本監控 | 中 | Phase 3 |

---

### 不做清單（明確排除）

以下功能在可見的未來不會開發，避免範圍蔓延：

- 直接在 Phoenix 內設計圖片（不做 Canva 的事）
- 自動發布到 Instagram（Meta API 限制多，維護成本高）
- 多語言支援（專注繁體中文市場）
- 團隊協作功能（先驗證單人使用者價值）
- 影片生成（技術成本與複雜度過高）
