# User Flow
## Phoenix — 使用者每日體驗設計

---

### 整體哲學

Phoenix 的使用者旅程分兩個層次：

- **首次使用（Onboarding）**：建立品牌記憶，Phoenix 才能主動推薦
- **每日使用（Daily Loop）**：打開 → 看推薦 → 選擇 → 產出 → 發布

目標是讓「每日使用」的決策時間控制在 **3 分鐘以內**。

---

### 首次使用：品牌建立（一次性）

```
Step 1｜你是誰？
  - 職業身份（保險業務、理財規劃師、講師...）
  - 主要服務（壽險、投資型、增員、退休規劃...）
  - 目前在哪個平台發文（Instagram / LINE / 兩者）
  - 目標受眾描述（30-45 歲上班族、女性為主、剛成家的族群...）

Step 2｜你的品牌聲音是什麼？
  - 選擇語氣風格（專業嚴謹 / 親切溫暖 / 幽默故事型）
  - 上傳 3-5 篇你喜歡的過去貼文（AI 分析你的寫作風格）
  - 你的核心信念是什麼？（一句話說你為什麼做這份工作）

Step 3｜過去的內容（可選）
  - 連結 Instagram 帳號（讀取近期貼文分析互動率）
  - 或手動輸入幾個「過去做過的主題」

Step 4｜目標設定
  - 你最想達成什麼？（開發新客戶 / 增員 / 建立信任感 / 全部）
  - 每週希望發幾次文？（1次 / 3次 / 5次）
```

完成後，Phoenix 生成「品牌身份卡」讓使用者確認，並存入品牌記憶庫。

---

### 每日使用：Daily Loop

```
早上打開 Phoenix（8:00 AM 推播通知）
│
▼
【Today's Brief】
  Phoenix 顯示今天的內容簡報：
  - 本週產業熱門話題 1 則
  - 你的受眾最近關注什麼（趨勢信號）
  - 你上次發文的效果摘要
│
▼
【今日推薦主題】（3 個選項）
  每個選項包含：
  - 主題標題
  - 推薦理由（"這週退休規劃在台灣搜尋量上升 40%"）
  - 預估適合的格式（輪播 / 單張 / Reels）
  - 預估互動潛力（高 / 中）
│
▼
使用者選擇其中一個（或修改主題）
│
▼
【內容產生】
  Phoenix 根據品牌記憶 + 選定主題自動產生：
  - 輪播封面 + 每頁文案
  - Instagram Caption
  - Hashtag（個人化，不是通用清單）
  - Canva 版型文字
  - 最佳發布時間建議
│
▼
使用者預覽 → 微調 → 複製到 Canva / 排程工具
│
▼
標記「已發布」→ Phoenix 記錄這篇的類型、格式、時間
│
▼
下次推薦時，Phoenix 已更新你的內容歷史
```

---

### 主要頁面結構

```
/                    → 首頁：Today's Brief + 推薦主題
/brand               → 品牌設定（身份、聲音、目標）
/generate/[id]       → 內容產生工作區（從推薦主題展開）
/history             → 過去產生的內容清單
/insights            → 內容效果分析（哪類主題最有共鳴）
```

---

### 關鍵 UX 決策

**推薦而非搜尋**
首頁不是空白輸入框，而是 3 張主題推薦卡片。使用者最多只需要點一下。

**理由透明**
每個推薦都附上一句「為什麼推這個」。使用者不是盲目信任 AI，而是看到數據後做決定。

**永遠有「自訂」出口**
如果不喜歡三個推薦，仍可手動輸入任何主題。Phoenix 的推薦是起點，不是限制。

**發布追蹤閉環**
使用者標記發布後，Phoenix 才有辦法學習。這個動作要極度簡單（一鍵）。

---

### 情境場景

**場景 A：業務員小明，早上 8:15**
打開 Phoenix，看到推薦主題「年金保險 vs 儲蓄險，傻傻分不清楚？」
推薦理由：「這週你的受眾中有 3 人在 IG 上分享了相關貼文，且你已經 2 週沒發理財知識類內容」
小明點下去，30 秒後輪播文案出現，複製到 Canva，設定排程，完成。

**場景 B：業務員小美，臨時想發文**
今天突然有一個真實客戶案例想分享。打開 Phoenix，選「自訂主題」，輸入「一個朋友 40 歲才開始規劃退休」，Phoenix 根據她的品牌聲音（溫暖故事型）產生有情感的輪播文。
